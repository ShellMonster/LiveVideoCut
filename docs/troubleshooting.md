# 问题排查指南

遇到问题时，先根据下面的症状描述找到对应条目，按步骤操作即可。大部分问题不需要查看日志，在界面上就能解决。

如果本指南没有覆盖你的问题，可以去诊断页面查看详细信息。每个任务都有独立的诊断面板，入口在任务队列页面点击任务卡片上的"诊断"图标。

## 上传问题

### 文件上传失败，提示"文件过大"

**症状：** 拖拽视频到上传区域后，进度条卡住或直接报错。

**可能原因：** 视频文件超过了上传限制。

**解决步骤：**

1. 检查视频文件大小。系统默认限制由 Docker Compose 配置决定，通常在 10GB 以内都能上传。
2. 如果视频确实很大（比如超过 2 小时的直播），建议先用视频剪辑工具截取需要的片段再上传，这样处理速度也更快。
3. 网络不稳定也会导致上传中断。如果上传到一半失败，重新上传即可，不会产生重复任务。

### 提示"不支持的视频格式"

**症状：** 上传后立即报错，提示格式不对。

**可能原因：** 视频不是 MP4 格式，或者编码不是 H.264 + AAC。

**解决步骤：**

1. 确认文件扩展名是 `.mp4`。
2. 有些 MP4 文件内部编码可能是 HEVC (H.265) 而非 H.264，系统不支持这种编码。
3. 用格式转换工具（如格式工厂、HandBrake）把视频转为 H.264 编码的 MP4。
4. 如果是手机录屏文件，iPhone 默认就是 H.264 + AAC 的 MP4，通常没问题。安卓手机可能需要检查录制设置。

### 提示"视频没有音频流"

**症状：** 上传报错说找不到音频。

**可能原因：** 视频是纯画面没有声音，或者音频编码不是 AAC。

**解决步骤：**

1. 用播放器打开视频，确认有没有声音。如果本身就没有声音，那这个视频不适合用本工具处理（字幕依赖语音转写）。
2. 如果视频有声音但仍然报错，可能是音频编码格式不对。用格式转换工具把音频轨转为 AAC 编码。
3. 某些直播回放会有音频轨缺失的情况，需要联系直播平台重新下载。

### 上传一直转圈，超时无响应

**症状：** 上传进度条走到某个位置后卡住不动。

**可能原因：** 网络问题或后端服务未正常启动。

**解决步骤：**

1. 先确认后端服务在运行。打开浏览器访问 `http://127.0.0.1:5538/health`，如果返回正常说明后端没问题。
2. 如果健康检查也打不开，用终端执行 `docker compose ps` 查看容器状态，如果有容器没启动，执行 `docker compose up -d` 重启。
3. 如果容器都在运行但上传还是卡住，查看后端日志：`docker compose logs api`，看有没有错误信息。
4. 大文件上传需要稳定的网络连接。如果通过远程服务器部署，确保带宽足够。

## 处理卡住或失败

### 卡在"抽帧检测"阶段不动

**症状：** 任务状态显示"处理中"，进度一直停在第一阶段。

**可能原因：** 视频文件异常导致 FFmpeg 抽帧失败，或者 Worker 内存不足。

**解决步骤：**

1. 先等待 5 分钟。长视频的抽帧检测阶段本身就需要几分钟，不是卡住了。
2. 如果超过 15 分钟还没动，去诊断页面查看事件日志，看有没有"内存不足"或"FFmpeg"相关的错误。
3. 确认 Docker 分配的内存是否足够。Worker 容器建议至少 4GB，宿主机至少 16GB 内存。在 Docker Desktop 设置里调整资源分配。
4. 如果视频分辨率特别高（比如 4K），处理时间会显著变长。可以先降低导出分辨率为 1080p。

### VLM 确认阶段报错

**症状：** 任务在第二阶段失败，错误信息提到 VLM 或 API。

**可能原因：** VLM API Key 无效、过期，或者模型名称填错了。

**解决步骤：**

1. 打开设置页的 AI 服务页签，检查 VLM API Key 是否正确。注意 Key 前后不要有空格。
2. 确认 API Base 地址是否正确。如果你用的是代理地址，确保能正常访问。
3. 检查模型名称是否正确。Qwen 系列通常类似 `qwen-vl-max`，GLM 系列类似 `glm-4v-plus`。具体名称参考各平台的文档。
4. 如果不需要 VLM，可以切换导出模式为"跳过 VLM"来绕过这个问题。

### ASR 转写超时或失败

**症状：** 第三阶段报错，提示转写失败或超时。

**可能原因：** ASR API Key 无效、网络不通、或者 ASR 服务配额用完了。

**解决步骤：**

1. 检查设置页转写服务页签里的 API Key 是否正确。
2. 如果你用的是火山引擎 ASR，还需要配置 TOS 存储相关参数（AK、SK、Bucket、Region、Endpoint），这些缺一不可。
3. 检查 ASR 服务的免费额度是否已用完。火山引擎各 ASR 服务各有 20 小时免费额度。
4. 如果暂时无法解决 ASR 问题，可以先关闭字幕（字幕模式选"关闭"），这样处理管线会跳过 ASR 阶段，至少能拿到切片结果。

### Worker 内存不足（OOM）

**症状：** 任务突然失败，日志里出现"Killed"或"Out of Memory"。

**可能原因：** Worker 容器内存不够。4K 视频或超长视频处理时内存消耗较大。

**解决步骤：**

1. 确认 Docker 给 Worker 容器分配了至少 4GB 内存。检查 `docker-compose.yml` 中 Worker 的 `mem_limit` 设置。
2. 降低导出分辨率为 1080p，减少 FFmpeg 处理的内存压力。
3. 如果视频很长（超过 1 小时），考虑截取片段分段处理。
4. 宿主机本身也需要足够内存。8GB 内存的服务器建议只同时处理一个任务。

## 切片结果不理想

### 切出来的片段太多了

**症状：** 20 分钟的直播切出了二三十个片段，很多都是重复的或没意义的。

**可能原因：** 换衣检测灵敏度太高，或者导出模式选了"全量候选"/"全量场景"。

**解决步骤：**

1. 确认导出模式是"智能"（有 VLM 确认过滤），不要选"全量候选"或"全量场景"，那两个是调试用的。
2. 在设置页的切分策略里，把切分粒度改为"整套搭配"，这样一套搭配只产出一个片段。
3. 如果检测灵敏度偏高，可以在高级参数里降低灵敏度（但当前默认设置对大多数直播已经比较合理了）。
4. 多余的片段可以在审核页面手动标记为"跳过"，不影响最终结果。

### 切出来的片段太少了

**症状：** 明明换了好多套衣服，但只切出两三个片段。

**可能原因：** 检测灵敏度太低，或者 VLM 过滤得太激进。也可能是视频画面变化不明显（比如背景太暗、主播动作太小）。

**解决步骤：**

1. 先用"全量候选"模式跑一次，看视觉检测到底识别出了多少个候选。如果候选也很少，说明是检测阶段的问题。
2. 在高级参数里提高换衣检测灵敏度。
3. 确认视频画质是否足够清晰。模糊的视频画面会导致检测器无法识别服装变化。
4. 如果视频里主播确实没有明显换衣动作（比如只换配饰），当前系统可能检测不到。这种情况可以试试开启 LLM 文本分析，通过语音内容辅助识别换品边界。

### 切片位置不对，开头或结尾被截断了

**症状：** 某个片段的开头缺了几个字，或者结尾突然断了。

**可能原因：** 切片边界没有对齐到完整的句子。

**解决步骤：**

1. "句边界对齐"功能默认是开启的，会自动把片段起止对齐到 ASR 句子边界。如果你之前手动关闭过，去设置页的切分策略里重新开启。
2. 开启"LLM 边界精修"（需要同时开启 LLM 文本分析），让 AI 审查每个片段的起止边界，确保开头完整、结尾自然。
3. 如果问题仍然存在，可以在审核页面手动调整片段的起止时间，然后重导出。

### 出现重复的片段

**症状：** 两个片段内容几乎一模一样，商品名也相同。

**可能原因：** 换衣检测在同一件商品附近触发了两次，或者视觉候选点太密集。

**解决步骤：**

1. 系统内置了片段去重机制，如果仍然出现重复，可以在审核页面手动跳过多余的片段。
2. 确认主播是否确实在同一段里展示了两次同一件商品（比如拿起来展示又放回去再拿起来）。这种情况两个片段可能都有意义。
3. 如果重复问题频繁出现，可以在高级参数里切换换衣检测融合模式为"加权投票"，减少误触发。

## 字幕问题

### 导出的视频没有字幕

**症状：** 明明设置了字幕模式，但视频里看不到字幕。

**可能原因：** 字幕模式实际设为"关闭"，或者 ASR 转写失败了。

**解决步骤：**

1. 先检查设置页的字幕模式是不是选了"关闭"。上传时选择的预设可能覆盖了你的设置。
2. 去诊断页面查看 ASR 阶段是否成功完成。如果 ASR 失败了，系统会跳过字幕烧录继续导出无字幕的视频。
3. 确认视频是否有语音。如果某段画面里主播没说话，那这段自然没有字幕。
4. 字幕可能太小或位置太靠边，被视频画面内容遮挡了。试试增大字号。

### 字幕和语音不同步

**症状：** 字幕显示的内容和当前语音对不上，要么快了要么慢了。

**可能原因：** ASR 时间戳不准确。

**解决步骤：**

1. 如果你用的是阿里 DashScope ASR，时间戳精度本身就不高（逐字时间戳是均匀分配的）。切换为"火山 VC 字幕"可以获得更准确的时间戳。
2. 如果已经是火山 VC 字幕但仍然不同步，可能是视频本身有音频偏移（直播录制的常见问题）。
3. 视频变速功能会同时调整字幕时序，正常不应该出现不同步。如果变速后不同步，可以在审核页面手动调整字幕时间并重导出。

### 字幕太小看不清

**症状：** 字幕在手机上看特别小，几乎读不出来。

**可能原因：** 字号设置偏小，或者视频分辨率较高导致字幕相对变小。

**解决步骤：**

1. 去设置页的字幕设置里调大字号。普通字幕默认 60，手机观看建议调到 70-80。
2. karaoke 高亮层字号默认 72，可以调到 80-90 让高亮字更醒目。
3. 字幕位置选"底部"通常在手机上最清晰，因为底部区域不容易被其他内容遮挡。
4. 如果导出分辨率是 4K，同样的字号在 4K 画面中会比 1080p 小一半。建议 4K 视频把字号调到 100 以上。

### Karaoke 跳字不自然，有跳跃感

**症状：** karaoke 模式下，高亮字跳动节奏不均匀，像在机械地跳。

**可能原因：** ASR 服务不是火山 VC 字幕。

**解决步骤：**

1. 这几乎肯定是 ASR 选错了。karaoke 字幕必须用"火山 VC 字幕"才能获得真实语音节奏的时间戳。
2. 阿里 DashScope 的逐字时间戳是均匀分配的假时间戳（每个字固定约 0.27 秒），用这个做 karaoke 跳字会非常机械。切换 ASR 后重新上传任务即可。
3. 火山 BigModel 标准版的时间戳是真实的，但分句不够智能，可能出现一个字独占一行的情况。还是推荐火山 VC 字幕。

## 下载和导出问题

### 片段下载失败

**症状：** 点击下载按钮后没反应，或者下载的文件打不开。

**可能原因：** 任务还没处理完，或者导出过程中出错了。

**解决步骤：**

1. 确认任务状态是"已完成"，不是"处理中"或"失败"。
2. 如果是批量下载失败，检查选中的片段是否超过 20 个。单次批量下载最多 20 个片段。
3. 刷新页面后重试。如果仍然失败，去诊断页面查看导出阶段是否有错误。
4. 尝试单个片段下载，排除是批量打包的问题。

### 视频片段无法播放

**症状：** 下载的 MP4 文件在播放器里打不开，或者只有画面没有声音。

**可能原因：** 导出过程被中断，文件不完整。

**解决步骤：**

1. 检查文件大小。正常的片段文件至少几百 KB，如果只有几 KB 说明导出失败了。
2. 在审核页面点击"重导出"重新生成这个片段。
3. 播放器兼容性问题。推荐用 VLC 播放器测试，它对各种编码格式的兼容性最好。
4. 如果只有声音没画面，可能是 FFmpeg 编码出了问题。尝试在设置里把 FFmpeg preset 从"fast"改为"medium"。

### 导出画质不满意

**症状：** 导出的片段画质比原视频差很多，有模糊或色块。

**可能原因：** FFmpeg 编码参数压缩过度。

**解决步骤：**

1. 在设置页的"高级参数"里调整 CRF 值。默认 23 是速度和画质的平衡，想要更好的画质可以调到 18-20（数值越小画质越好，但文件更大）。
2. 把 FFmpeg preset 从"fast"改为"medium"，编码更精细，画质会好一些，但处理时间会变长。
3. 确认导出分辨率设置。如果原视频是 4K 但导出分辨率选了 1080p，确实会降质。选"原始分辨率"可以保持原画质。
4. 注意：短视频平台上传后还会二次压缩，所以导出时画质尽量保持高一点。

## 诊断工具

当以上方法都无法解决你的问题时，可以使用系统内置的诊断工具进一步排查。

### 诊断页面

每个任务都有独立的诊断面板，点击任务队列页面中任务卡片上的诊断图标即可进入。诊断页面包含：

- **指标卡片**：展示处理时长、片段数量、成功率等关键指标。
- **管线阶段**：横向展示四个阶段的耗时，一眼看出哪个阶段耗时最长。
- **漏斗图**：展示从候选到最终导出的数量变化，帮你判断过滤是否合理。
- **事件日志**：按时间排列的所有处理事件，包括警告和错误，支持分页浏览。
- **异常建议**：系统会自动分析常见异常并给出修复建议。

### 下载调试包

如果需要联系开发者反馈问题，可以下载完整的调试包（artifacts.zip）。在诊断页面点击"导出报告"按钮，系统会打包该任务的所有配置、日志、分段信息和错误详情为一个 ZIP 文件。把调试包和问题截图一起发给开发者，可以大幅加快问题定位速度。
