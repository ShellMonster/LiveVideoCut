# 术语表

使用过程中你可能会碰到一些技术名词。这个页面用大白话解释每个词是什么意思，以及跟你有什么关系。

---

## ASS

一种字幕文件格式，支持丰富的排版效果，比如字体颜色、描边、动画。

跟普通字幕相比，ASS 格式能做到"逐字变色"、"逐字放大"这样的特效。karaoke（卡拉OK）字幕用的就是 ASS 格式，所以才能实现一个字一个字跟着声音高亮的效果。

**对你有什么影响**：选了 karaoke 字幕模式，系统会自动生成 ASS 文件，你不需要手动处理。如果你从 clips 目录拷出字幕文件做二次加工，需要用支持 ASS 格式的播放器或剪辑软件打开。

---

## ASR（语音转写）

ASR 全称是 Automatic Speech Recognition，就是把视频里说的话自动转成文字。

你可以把它理解成一个"超快的打字员"，不仅能听写内容，还能精确记录每个字出现在视频的哪一秒。转写出来的文字带时间戳，后续才能把字幕对准声音。

**对你有什么影响**：ASR 的质量直接影响字幕的准确度和 karaoke 跳字的同步性。推荐选"火山 VC 字幕"，效果最好。

---

## CRF

CRF 全称是 Constant Rate Factor，是控制视频画质的一个数字参数。

简单说：CRF 数字越小，画质越好，但文件越大；数字越大，画质越差，但文件越小。默认值是 23，这个值在画质和体积之间取得了不错的平衡。

**对你有什么影响**：如果你觉得导出的片段画质不够清晰，可以在设置页的高级参数里把 CRF 调小（比如调到 18 或 20）。文件会变大，但画面更清晰。反过来，如果嫌文件太大，可以调到 26 或 28。

---

## Docker

Docker 是一种"打包运行环境"的工具。它把应用和所有依赖项装在一个"容器"里，确保在任何电脑上都能一致地运行。

打个比方，Docker 就像是一个预装好所有软件的虚拟电脑。你不需要自己安装 Python、FFmpeg、Redis 这些东西，Docker 帮你全准备好了。

**对你有什么影响**：启动这个系统只需要一条命令（`docker compose up -d`），不需要手动配置任何开发环境。前提是你的电脑已经安装了 Docker Desktop。

---

## FFmpeg

FFmpeg 是一个专业的视频处理工具，负责视频的剪切、拼接、烧字幕、加背景音乐等所有"画面加工"操作。

你可以把它当成系统里的"视频剪辑师"。抽帧、切片段、烧字幕、加变速、混音乐，这些最终改变视频画面的活，全是 FFmpeg 在干。

**对你有什么影响**：你在设置页调的画质（CRF）、编码速度（preset）、视频倍速等参数，最终都由 FFmpeg 执行。这些参数影响导出速度和成品画质。

---

## karaoke（卡拉OK字幕）

karaoke 模式的字幕会像 KTV 歌词一样，随着声音逐字高亮，当前正在读的字还会弹跳放大。

跟普通白字字幕比，karaoke 字幕更抓眼球，特别适合短视频平台（抖音、小红书等）。观众一看就知道主播在说什么，阅读体验更好。

**对你有什么影响**：选 karaoke 模式需要配合"火山 VC 字幕"ASR 才能获得最佳效果。其他 ASR 的逐字时间戳不够精确，跳字会不同步。karaoke 字幕的处理时间比普通字幕略长。

---

## LLM（大语言模型）

LLM 全称是 Large Language Model，就是 ChatGPT、文心一言这类 AI。在这个系统里，LLM 负责"读懂文字"。

它的工作是分析 ASR 转写出来的文字内容，判断主播什么时候开始讲一件新的商品。比如主播从讲毛衣切换到讲裙子，LLM 能从文字里识别出这个切换点。

**对你有什么影响**：LLM 分析默认是关闭的。如果你发现纯靠画面检测切分不够准确（比如主播讲了多件商品但没换衣服），可以开启 LLM 分析，它会通过文字内容辅助判断切分点。开启需要配置 LLM API Key。

---

## SRT

SRT 是最常见、最通用的字幕文件格式。几乎所有播放器和剪辑软件都支持。

SRT 格式很简单：每一句字幕包含序号、起止时间和文字内容，没有花哨的排版效果。适合只需要基础字幕的场景。

**对你有什么影响**：选了 basic 或 styled 字幕模式，系统会生成 SRT 文件。SRT 文件兼容性最好，你可以把它导入到剪映、PR 等剪辑软件里做二次编辑。

---

## VLM（视觉语言模型）

VLM 全称是 Vision Language Model，是一种能"看图说话"的 AI。它像一个特别靠谱的助手，你给它看一张直播画面，它能告诉你画面里是什么商品、什么颜色、什么款式。

在这个系统里，VLM 负责对换衣检测找到的候选片段做二次确认。它会逐帧分析画面，判断这个片段是不是真的在讲一件新商品，过滤掉误判的情况。

**对你有什么影响**：开启 VLM（选"智能"导出模式）可以显著减少无效片段，但需要配置 VLM API Key，会产生少量费用。如果不想花钱或没有 API Key，可以选"跳过 VLM"模式。

---

## YOLO

YOLO 是一种 AI 物体检测模型，全称是 "You Only Look Once"。它能快速识别画面里有哪些物体、在什么位置。

在这个系统里，YOLO 被用来识别直播画面中的服装品类（上衣、裙子、裤子等）。它就像一双能自动分类的"电子眼"，帮系统判断主播换了什么衣服。

**对你有什么影响**：YOLO 的检测阈值（默认 0.25）在设置页的高级参数里可以调整。降低阈值会更敏感，可能多切出一些片段；提高阈值会更严格，减少误切。

---

## 导出模式

导出模式决定了 AI 参与剪辑的程度，一共四种：

| 模式 | AI 参与程度 | 效果 |
|------|-----------|------|
| 智能模式（smart） | VLM 会逐帧确认 | 只保留真正在讲商品的片段，精度最高 |
| 跳过 VLM（no_vlm） | 不调用 VLM | 按画面检测结果直接切，速度快、不花钱 |
| 全量候选（all_candidates） | 不调用 VLM | 导出所有检测到的候选，片段可能很多 |
| 全量场景（all_scenes） | 不调用 VLM | 按场景变化切割，片段最多 |

**对你有什么影响**：日常使用选"智能"模式效果最好。"跳过 VLM"适合没有 API Key 或想省钱的时候。"全量候选"和"全量场景"是调试用的，帮你看看系统到底检测到了什么。

---

## 换衣检测

换衣检测是系统自动找出主播换衣服的时间点，以此作为切分视频的依据。

系统不只是简单地比较画面差异。它同时看五个维度：服装品类有没有变（YOLO）、服装区域像素有没有变（MediaPipe）、整体颜色有没有变（HSV）、上下身颜色各自有没有变（分区域 HSV）、纹理有没有变（ORB）。五路信号综合判断，比单纯看画面变化准确得多。

**对你有什么影响**：换衣检测的灵敏度可以在设置页调整。如果你觉得切多了（比如主播只是拿了一下衣服就被当成换衣了），可以提高检测灵敏度到"保守"模式；如果觉得切少了，可以调成"敏感"模式。

---

## 句边界对齐

句边界对齐是把切分点对齐到完整句子的技术，避免把一句话切成两半。

举个例子：如果系统检测到主播在第 35.7 秒换了衣服，但这一秒刚好在句子"这件毛衣的面料是"的中间，句边界对齐会把切分点挪到这句话说完的位置，确保每个片段开头都是一句完整的话。

**对你有什么影响**：这个功能默认开启，你不需要手动操作。它确保每个片段的视频开头不会出现"半句话"的情况，观看体验更自然。

---

## 信号融合

信号融合是把"画面上看到的"和"文字里说的"两种信息合并起来，决定最终在哪里切分。

画面检测告诉你主播什么时候换了衣服，文字分析告诉你主播什么时候开始讲新商品。两路信号有时候不完全一致。信号融合会用一个"两层树"结构把它们对齐：先按换衣节点把视频分成大段，再在大段里按商品讨论的切换点细分。

**对你有什么影响**：信号融合需要同时开启视觉检测和 LLM 文本分析。如果你只用了视觉检测（没开 LLM），系统会用单层模式，只按换衣节点切分，效果也不错但可能不够精细。

---

## 预设

预设是一组配置参数的快捷组合，帮你省去逐个调参数的麻烦。

系统内置了 4 种预设，每种预设都针对特定使用场景优化了所有参数（导出模式、字幕样式、ASR 引擎等）。你选一个预设，所有参数自动填好，直接上传就行。

**对你有什么影响**：新手直接用预设就够了，不需要理解每个参数的含义。等你对系统熟悉了，可以在设置页微调单个参数，或者上传时不选预设、完全自定义。
